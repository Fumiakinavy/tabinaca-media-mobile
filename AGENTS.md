# プロジェクト指示（Tabinaca iOS）

## 目的
- `context/tabinaka-media-staging-develop` にあるWebサイトのiOS版を作る。
- `context/` 内のWebコードをUI/UX・挙動の一次情報とする。

## 絶対ルール
- `context/` 配下は読み取り専用。編集・整形・ファイル生成を一切しない。
- iOS実装は `tabinaca-media-ios/` など `context/` 以外の場所で行う。

## 必須フロー
1. `context/` のWebプロジェクトを読み、仕様を理解する。
2. 実装前に `plan/` にMarkdownの計画書を作成する（例: `plan/plan.md`）。
   - 目的/スコープ
   - 画面一覧と画面遷移
   - データソース/API
   - アーキテクチャ選定（SwiftUI / UIKit など）
   - マイルストーン/タスク
   - 不明点・リスク
3. 計画書の作成後にのみ、`tabinaca-media-ios/` で実装を開始する。
4. 途中で要件が変わったら、まず `plan/` を更新してから実装を続ける。

## 実装方針
- Webのデザイン/情報設計/インタラクションを可能な限り忠実に再現する。
- 必要なアセットは `context/` からコピーしてよいが、元ファイルは編集しない。
- 変更は小さく分割し、重要な判断は計画書に記録する。

## コミュニケーション
- 進捗・判断は日本語で簡潔に報告する。
- 不明点は短く質問する。
